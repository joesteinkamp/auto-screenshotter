# Auto Screenshotter MCP Relay

A tiny local bridge so AI tools (Claude.ai custom connectors, Cursor, etc.) can
drive the Auto Screenshotter Chrome extension over MCP. The relay is a protocol
bridge only — zip files never transit this process; they're saved by the
extension directly to your browser's default Downloads folder.

```
 AI client ──HTTP/SSE──▶ relay (this) ◀──WebSocket── extension
```

## Run

```bash
cd relay
npm install
npm run dev      # tsx watch, default port 3848
```

Change the port with `PORT=3849 npm run dev`. The relay binds to `127.0.0.1`
by default; set `HOST=0.0.0.0` to expose it (not recommended — no auth).

## Endpoints

| Method | Path               | Purpose                                  |
| ------ | ------------------ | ---------------------------------------- |
| POST   | `/mcp/:userId`     | JSON-RPC MCP requests                    |
| GET    | `/mcp/:userId`     | SSE channel for server events            |
| GET    | `/healthz`         | Liveness                                 |
| WS     | `/ws?id=:userId`   | Extension tunnel                         |

`:userId` is an 8-char id generated by the extension on first run and shown
in the side panel's **MCP** tab.

## Tools

- `screenshot_urls({ urls[], useLlm?, scrollBehavior? })`
- `crawl_site({ startUrl, maxPages?, maxDepth?, useLlm?, sameOriginOnly?, scrollBehavior? })`
- `get_job_status({ jobId?, waitForChange?, sinceTs? })`

Capture tools return immediately with a job id. Poll `get_job_status` (with
`waitForChange: true` for long-polling up to 60s) to track progress. When the
job completes, the text result includes the path of the zip inside the user's
Downloads folder.

## Sanity check with curl

```bash
# tools/list
curl -s -X POST http://localhost:3848/mcp/YOURID \
  -H 'content-type: application/json' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}' | jq

# start a screenshot job (must have the extension connected)
curl -s -X POST http://localhost:3848/mcp/YOURID \
  -H 'content-type: application/json' \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"screenshot_urls","arguments":{"urls":["https://anthropic.com"]}}}' | jq
```

## Client configuration

### Clients that speak HTTP+SSE MCP directly

Point them at `http://localhost:3848/mcp/YOURID`.

### Claude Desktop

Claude Desktop currently launches MCP servers over stdio. Until a stdio shim
ships, use a client that supports HTTP MCP (Cursor, claude.ai custom
connectors, or your own proxy).

## Security

The default bind is `127.0.0.1` and there's no auth token — this is a
single-user localhost setup. If you ever expose it beyond localhost, add a
shared-secret check first.
